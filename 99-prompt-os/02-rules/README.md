# SPOS Rules

> Status: Folder contract Draft SPOS-001; rule substantif belum dibuat atau approved.

## Tujuan

Menjadi lokasi kanonik bagi rule eksekusi AI yang atomik, dapat dikomposisi, memiliki precedence, dan dapat ditelusuri ke authority upstream.

## Kontrak Rule Minimum

Setiap rule masa depan wajib menyatakan:

- identifier dan title;
- status serta versi;
- owner, reviewer, dan approver;
- sumber authority dan dependency;
- scope, constraint, exception, serta stop condition;
- validation evidence dan review trigger;
- relasi ke rule yang superseded atau terkait.

## Boundary

Rule SPOS mengoperasionalkan policy yang sah. Folder ini tidak menggantikan governance Foundation, standar repository, keputusan domain, atau sumber Kernel. Prompt dan playbook harus merujuk rule kanonik, bukan membuat salinan rule.
