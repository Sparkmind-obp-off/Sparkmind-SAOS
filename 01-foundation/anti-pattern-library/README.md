# Anti-pattern Library

> Status: Struktur aktif; anti-pattern substantif belum dikurasi.

## Tujuan

Mendokumentasikan praktik berulang yang tampak membantu tetapi menghasilkan risiko, kegagalan, atau biaya jangka panjang, beserta cara mendeteksi dan memulihkannya.

## Ruang Lingkup

Mencakup anti-pattern organisasi, dokumentasi, governance, knowledge, dan decision. Tidak digunakan untuk menyalahkan individu atau menyimpan incident detail sensitif.

## Jenis Dokumen

- anti-pattern description;
- symptom dan detection signal;
- root condition dan consequence;
- prevention, mitigation, dan recovery guidance.

## Hubungan

- Bersumber dari outcomes, incident review, [`../decision-library/`](../decision-library/README.md), dan [`../wisdom/`](../wisdom/README.md).
- Menjadi pembanding bagi [`../pattern-library/`](../pattern-library/README.md).
- Mitigasi approved dapat masuk ke [`../playbooks/`](../playbooks/README.md).

## Guardrail

Dokumen harus fokus pada sistem dan kondisi, meminimalkan data sensitif, serta menyertakan evidence dan batas generalisasi.
