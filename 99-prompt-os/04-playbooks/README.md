# SPOS Playbooks

> Status: Folder contract Draft SPOS-001; playbook substantif belum dibuat atau approved.

## Tujuan

Menyimpan prosedur terurut untuk kelas pekerjaan berulang dengan precondition, input, rule dependency, langkah, quality gate, rollback, dan escalation yang eksplisit.

## Aturan

- Playbook diturunkan dari governance, keputusan, pattern, dan rule yang statusnya sesuai.
- Playbook tidak menciptakan authority baru.
- Setiap playbook wajib menyatakan kapan digunakan dan kapan tidak digunakan.
- Perbedaan dari [`../../01-foundation/playbooks/`](../../01-foundation/playbooks/README.md): Foundation memiliki pengetahuan prosedural lintas domain; SPOS memiliki orchestration contract khusus eksekusi AI dan harus merujuk playbook Foundation bila menjadi sumbernya.
